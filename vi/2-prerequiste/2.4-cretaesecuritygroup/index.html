Tạo Security Group

Tương tự như một thành phần Firewall trong mạng LAN, Security Group quản lý quyền truy cập vào các thành phần trong một Virtual Private Cloud (VPC).

Truy cập vào VPC > Security Groups > Create security group để tạo 1 SG

  1. Internet Facing Load Blancer Security Group

    Đối với Internet Facing Load Blancer, Security Group sẽ cho phép truy cập HTTP vào cổng 80 từ Internet vào Load Blancer của Web Tier.

    • Security Group name, nhập tên Internet-Facing-LB-SG
    • Description, nhập miêu tả
    • VPC, chọn FCJ-WS-1 VPC.
    • Thêm Inbound rules cho phép My IP truy cập HTTP vào từ cổng 80

  1. Bastion Host Security Group

    Bastion Host cho phép SSH từ Local

    • Security Group name, nhập tên Bastion-SG
    • Description, nhập miêu tả
    • VPC, chọn FCJ-WS-1 VPC mà chúng ta đã tạo trước đó.
    • Thêm Inbound rules cho phép My IP truy cập SSH vào từ cổng 22

  1. Web Security Group

    Đối với các Public Instance trong tầng web, chúng ta cần tạo một Security Group cho phép lưu lượng từ cân bằng tải, SSH từ máy tính cục bộ để có thể cấu hình EC2, và một cổng HTTP từ địa chỉ IP của máy tính cục bộ để thử nghiệm hoạt động từ trang web.

    • Security Group name, nhập tên Web-SG
    • Description, nhập miêu tả
    • VPC, chọn FCJ-WS-1 VPC mà chúng ta đã tạo trước đó.
    • Thêm các Inbound rules
      1. Cho phép My IP truy cập SSH vào từ cổng 22.
      2. Cho phép HTTP từ My IP vào cổng 80.
      3. Cho phép HTTP từ Security Group “Internet-Facing-LB-SG” truy cập vào từ cổng 22.

  1. Internal Facing Load Balancer Security Group

    Nhóm bảo mật này được sử dụng để quản lý lưu lượng HTTP từ Web Security Group vào cổng 80. Điều này cho phép lưu lượng từ các instance trong Web Tier truy cập vào cân bằng tải nội bộ.

    • Security Group name, nhập tên Internal-LB-SG
    • Description, nhập miêu tả
    • VPC, chọn FCJ-WS-1
    • Thêm Inbound rules cho phép Web-SG truy cập HTTP vào từ cổng 80

  1. App Security Group

    Nhóm bảo mật này được sử dụng để quản lý lưu lượng HTTP từ Internal-LB-SG vào cổng 80, Bastion-SG SSH vào từ cổng 22.

    • Security Group name, nhập tên App-SG
    • Description, nhập miêu tả
    • VPC, chọn FCJ-WS-1
    • Thêm các Inbound rules 1. Cho phép Bastion-SG truy cập SSH vào từ cổng 22 2. Cho phép Internal-LB-SG TCP vào từ cổng 4000

  1. Database Security Group

    Nhóm bảo mật này cho phép EC2 trong App Tier truy cập vào MySQL Database qua cổng 3306.

    • Security Group name, nhập tên Database-SG
    • Description, nhập miêu tả
    • VPC, chọn FCJ-WS-1 VPC
    • Thêm Inbound rules cho phép App-SG truy cập vào từ cổng 3306
